# Prompt Engineer - Quality Checklist

## Pre-Generation Checklist
- [ ] GitHub issue URL is valid and accessible
- [ ] Issue content has been successfully fetched
- [ ] Project tech stack has been analyzed
- [ ] Architecture pattern has been identified
- [ ] Issue type has been classified correctly
- [ ] Complexity level is appropriate

## Prompt Content Checklist
- [ ] Project context section is complete
- [ ] Objective clearly states what needs to be achieved
- [ ] Requirements are extracted and listed
- [ ] Technical considerations are documented
- [ ] Edge cases are identified
- [ ] Agent workflow is appropriate for issue type
- [ ] Each phase has clear deliverables
- [ ] Quality standards match project conventions
- [ ] Testing criteria match selected priority
- [ ] Success criteria are measurable
- [ ] Branch naming follows conventions

## Agent Selection Validation
- [ ] Agent count matches complexity level
- [ ] UI agents included if visual changes required
- [ ] Architecture agent included if structural changes
- [ ] Security considerations if applicable
- [ ] Testing agent matches testing priority
- [ ] Code review is always final step

## Output Quality
- [ ] Prompt is self-contained and actionable
- [ ] Instructions are clear and unambiguous
- [ ] No placeholder text remaining
- [ ] Language matches project conventions
- [ ] Format is consistent and readable
